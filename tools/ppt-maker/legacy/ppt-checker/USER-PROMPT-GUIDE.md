# PPT Checker — Prompt Guide

## What this agent does

PPT Checker reviews SettleMint PowerPoint presentations and produces a structured markdown review report. It behaves like a senior brand and presentation reviewer — scoring the deck across 10 dimensions, naming specific weak slides, auditing for DALP terminology errors, and issuing a clear verdict: **Approve / Revise / Rebuild**.

Typical review output includes:
- Verdict: Approve / Revise / Rebuild
- Score summary across 10 dimensions (brand compliance, narrative flow, design quality, content accuracy, DALP terminology, editability, slide-level discipline, persuasion, final-deck readiness, overall fit for purpose)
- Hard failures (blockers that prevent Approve regardless of other scores)
- Top issues with slide-level specificity
- Recommended fix path
- Optional DOCX export of the review report

## When to use it

- Before sending any deck to a prospect, client, or investor
- After PPT Maker builds a deck — PPT Checker is its natural quality gate
- When an internally edited deck has gone through multiple rounds and you want an independent read
- When a deck has been adapted from another version and needs a fresh compliance check
- When you need a second opinion before a high-stakes presentation

## Required inputs

| Input | Details |
|-------|---------|
| **PPTX file path** | Full or relative path to the `.pptx` file. Common locations: `ppt-checker/input/`, `ppt-maker/output/`. Example: `ppt-checker/input/dalp-demo-v2.pptx` |

## Optional / configurable parameters

| Parameter | Default | Notes |
|-----------|---------|-------|
| **Context** | None | Describe the deck's purpose, audience, and intended use. This helps the reviewer calibrate expectations. E.g. "This is a first-meeting executive deck for a central bank." |
| **Specific concerns** | None | List any areas you already suspect are weak: e.g. `I'm not happy with the compliance section`, `the narrative flow feels off after slide 8`, `check the DALP terminology in the token features section` |
| **DOCX export** | Not generated by default | Request `export: docx` if you need the review report in Word format for internal sharing |
| **Evaluator focus** | Balanced review | Specify a dimension to prioritise: `brand`, `narrative`, `content-accuracy`, `dalp-terminology`, `persuasion` |

## Full example prompt

```
Please review this deck before we send it to the MAS team in Singapore.

Deck: ppt-maker/output/2026-03-20-mas-executive-brief.pptx

Context: 
- This is a first executive meeting with MAS leadership — CTO and Head of FinTech 
  Innovation. The goal is to get invited to a follow-on technical deep-dive.
- Audience is senior but technically literate; they have seen a lot of blockchain pitches.
- The deck was built by PPT Maker and then manually edited to add a Singapore-specific 
  regulatory slide.

Specific concerns:
- The manually added regulatory slide (slide 9) may not match the template style
- I want extra scrutiny on DALP terminology — confirm nothing sounds like a competitor product
- Check whether the narrative builds correctly toward the close

Export: docx
```

## ⚠️ Important

- **Send everything in ONE message.** Provide the deck path, context, specific concerns, and export preferences all at once.
- **Do not send follow-up messages mid-task.** The agent will confirm when the review report is ready.
- **The verdict is honest.** A Rebuild verdict means the deck should not be sent as-is. Act on it rather than asking the agent to soften the assessment.
- **Hard failures block Approve regardless of other scores.** A single brand violation, DALP terminology error on a critical claim, or broken template compliance can trigger a hard fail.
- **The PPTX must be a real PowerPoint file.** PDFs and exported images cannot be reviewed. If you only have a PDF, regenerate the PPTX from source first.
- **Review reports are saved to `ppt-checker/output/`.** Use descriptive filenames like `mas-brief-review.md` or `dalp-demo-v3-review.md`.
