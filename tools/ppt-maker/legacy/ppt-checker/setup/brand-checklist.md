# Brand Compliance Checklist

Use this as a **pass/fail review checklist** for any deck claiming to use the official SettleMint presentation template.

**Template baseline**
- Template size: **10.0" × 5.6"**
- Aspect ratio: **16:9 widescreen**
- Primary brand font: **Figtree**
- Content slide range: **5–28**
- Protected branded slides: **1, 2, 29, 30**

## 1) Typography

- [ ] **PASS if** every editable text object uses the **Figtree** font family only.
- [ ] **FAIL if** any text uses Arial, Calibri, Aptos, Helvetica, Montserrat, or any other non-Figtree font.
- [ ] **PASS if** only legitimate Figtree weights are used (Light, Regular, Medium, SemiBold, Bold, ExtraBold).
- [ ] **FAIL if** a fallback font appears anywhere in titles, body copy, captions, tables, footnotes, or page numbers.
- [ ] **PASS if** title text is visually dominant and clearly larger than body text on the same slide.
- [ ] **PASS if** body text on normal content slides is typically in the **16–24 pt** readable range.
- [ ] **FAIL if** body text drops below **14 pt** anywhere in the deck.
- [ ] **FAIL if** legal notes, labels, captions, or footnotes drop below **10 pt**.
- [ ] **PASS if** titles are typically in the **24–36 pt** range and remain consistent across similar slide types.
- [ ] **FAIL if** titles are so small that they read like body copy or so large that they break layout balance.
- [ ] **PASS if** text hierarchy is clear: title > section lead-in > body > caption/footnote.
- [ ] **FAIL if** too many font sizes create a messy, inconsistent hierarchy on a slide.
- [ ] **PASS if** line spacing and paragraph spacing make dense slides comfortably readable.
- [ ] **FAIL if** text is visibly cramped, overlapping, clipped, or running off the safe area.
- [ ] **PASS if** body copy uses sentence case or normal title case.
- [ ] **FAIL if** body paragraphs, bullets, or captions are written in **ALL CAPS**.
- [ ] **PASS if** ALL CAPS is used, at most, sparingly for short titles, labels, or micro-UI style tags.
- [ ] **FAIL if** bold, italics, underline, or color are overused for emphasis and create visual noise.

## 2) Color System

**Template theme palette (source: `ppt-maker/templates/template.pptx` → `ppt/theme/theme1.xml`)**
- Brand blue / Accent 1: **#4285F4**
- Near-black / Accent 2: **#212121**
- Slate / Accent 3: **#78909C**
- Orange accent / Accent 4: **#FFAB40**
- Teal accent / Accent 5: **#0097A7**
- Lime accent / Accent 6: **#EEFF41**
- Black / Dark 1: **#000000**
- White / Light 1: **#FFFFFF**
- Light neutral / Light 2: **#EEEEEE**
- Dark neutral / Dark 2: **#595959**

- [ ] **PASS if** the deck uses only the official SettleMint/template palette above.
- [ ] **FAIL if** arbitrary new colors are introduced for fills, text, charts, icons, borders, or callouts.
- [ ] **PASS if** content slides use **white (#FFFFFF)** as the default background.
- [ ] **FAIL if** content slides 5–28 use blue, grey, photo, texture, or colored backgrounds without a template-driven reason.
- [ ] **PASS if** dark text on white slides uses **#212121**, **#000000**, or approved neutral darks.
- [ ] **FAIL if** body text uses low-contrast pastel or decorative colors.
- [ ] **PASS if** brand blue (**#4285F4**) is used for branded emphasis, dividers, numbering, or key highlights.
- [ ] **PASS if** secondary accents (**#78909C, #FFAB40, #0097A7, #EEFF41**) are used sparingly and intentionally.
- [ ] **FAIL if** accent colors compete with each other or turn a slide into a rainbow.
- [ ] **PASS if** charts and diagrams use a restrained subset of brand colors with clear contrast.
- [ ] **FAIL if** chart series use random default PowerPoint colors outside the approved palette.
- [ ] **FAIL if** gradient backgrounds appear on content slides.
- [ ] **PASS if** branded or dark backgrounds are limited to cover, section-divider, and closing slides where the template already supports them.
- [ ] **FAIL if** text becomes hard to read because foreground/background contrast is too weak.

## 3) Logo & Brand Elements

- [ ] **PASS if** the SettleMint logo keeps its original proportions at all times.
- [ ] **FAIL if** the logo is stretched, squashed, rotated, skewed, cropped, or partially hidden.
- [ ] **FAIL if** the logo is recolored, opacity-reduced, outlined, shadowed, or stylized.
- [ ] **PASS if** the logo appears only in approved template positions or equivalent clean, intentional placements.
- [ ] **FAIL if** the logo is randomly moved, duplicated excessively, or used as decoration.
- [ ] **PASS if** the logo has enough clear space and does not collide with titles, edges, page numbers, or other graphics.
- [ ] **FAIL if** the logo touches slide edges or sits uncomfortably close to other elements.
- [ ] **PASS if** decorative brand shapes on slides **1, 2, 29, and 30** remain intact.
- [ ] **FAIL if** decorative brand shapes on slides **1, 2, 29, and 30** are deleted, recolored, resized, cropped, redrawn, or moved.
- [ ] **PASS if** any added icons or illustrations feel visually compatible with the template rather than fighting it.
- [ ] **FAIL if** homemade badges, stickers, glows, shadows, or gimmicky visual effects dilute the brand system.

## 4) Slide Structure

- [ ] **PASS if** the presentation remains in the official **10.0" × 5.6"** template size.
- [ ] **FAIL if** the deck has been resized to another aspect ratio or slide dimension.
- [ ] **PASS if** the deck starts from the official SettleMint template rather than an unrelated PowerPoint theme.
- [ ] **PASS if** cover and closing slides use branded backgrounds only where the template intends them.
- [ ] **PASS if** content slides **5–28** keep the expected clean white content background.
- [ ] **PASS if** page numbers are present and correctly placed on slides **5–28**.
- [ ] **FAIL if** page numbers are missing, duplicated, misaligned, or obviously broken on slides **5–28**.
- [ ] **PASS if** slide margins and alignment feel consistent from slide to slide.
- [ ] **FAIL if** objects drift off-grid, hug the edge, or ignore the template’s spacing logic.
- [ ] **PASS if** cover, divider, content, and closing slides each use layouts appropriate to their role.
- [ ] **FAIL if** a branded opening/closing layout is misused as a normal content slide.
- [ ] **PASS if** unused template slides are removed from the final delivered deck.
- [ ] **FAIL if** empty or placeholder template slides remain in the final deck.

## 5) Content Format

- [ ] **PASS if** all slide text is native PowerPoint text in editable text boxes, placeholders, tables, or chart labels.
- [ ] **FAIL if** any headline, paragraph, bullet list, table text, or label is baked into an image.
- [ ] **PASS if** charts are native PPTX chart objects or clearly editable vector/group objects.
- [ ] **FAIL if** charts are inserted as screenshots, exports, or flattened images.
- [ ] **PASS if** tables are native PPTX tables or editable shapes/text assemblies.
- [ ] **FAIL if** tables are screenshots, pasted snips, or non-editable image blocks.
- [ ] **PASS if** diagrams, labels, and callouts remain editable in PowerPoint.
- [ ] **FAIL if** screenshots are used where editable UI mockups, tables, or text should have been rebuilt natively.
- [ ] **PASS if** screenshots are used only for genuine product UI or evidence, not as a shortcut for layouting copy.
- [ ] **FAIL if** slide content is delivered as full-slide image exports masquerading as editable slides.

## 6) Technical Visual Integrity

### Text Overflow & Overlap
- [ ] **PASS if** no text is visibly clipped, cut off, or overflowing its textbox boundary.
- [ ] **FAIL if** any text is truncated, runs outside its container, or overlaps adjacent elements.
- [ ] **PASS if** auto-fit/shrink does not reduce text below minimum readable sizes (11pt body, 8pt caption).
- [ ] **FAIL if** auto-shrink makes any text unreadably small to avoid overflow.

### Element Overlap
- [ ] **PASS if** no content-bearing elements (text boxes, cards, images, shapes) visually overlap unintentionally.
- [ ] **FAIL if** cards, text boxes, or images overlap each other when they clearly should not.
- [ ] **PASS if** z-ordering is correct (no text hidden behind shapes, no labels buried under images).
- [ ] **FAIL if** an element that should be visible is obscured by another element on top of it.

### Font Sizing Consistency
- [ ] **PASS if** all slide titles use a consistent size (24pt ±2pt per brand system).
- [ ] **FAIL if** title sizes vary by more than 2pt across content slides without layout justification.
- [ ] **PASS if** body text is consistent at 11–14pt throughout the deck.
- [ ] **FAIL if** body text sizes vary arbitrarily across slides of the same type.
- [ ] **PASS if** visual hierarchy is clear: title > heading > body > caption on every slide.
- [ ] **FAIL if** titles are the same size as or smaller than body text on any slide.

### Element Spacing
- [ ] **PASS if** content stays within defined margins (minimum 0.50" from all edges, 0.93" preferred).
- [ ] **FAIL if** any content element is closer than 0.50" to a slide edge.
- [ ] **PASS if** gutters between columns, cards, or panels are consistent (±0.05" tolerance).
- [ ] **FAIL if** spacing between similar elements varies noticeably on the same slide.
- [ ] **PASS if** cards and panels do not touch each other or slide edges.
- [ ] **FAIL if** cards are jammed together with no visible gap between them.

### Title Placement
- [ ] **PASS if** all content slide titles are positioned in the title zone (Y: 0.47"–1.13" at scaled coordinates).
- [ ] **FAIL if** title position drifts by more than 0.10" across content slides.
- [ ] **PASS if** titles share consistent left alignment across content slides.
- [ ] **FAIL if** some content slide titles are left-aligned and others center-aligned without layout justification.

### Logo Integrity
- [ ] **PASS if** the SettleMint logo maintains its original aspect ratio wherever it appears.
- [ ] **FAIL if** the logo is stretched, squashed, rotated, or distorted in any way.
- [ ] **PASS if** the logo has at least 0.25" clear space on all sides.
- [ ] **FAIL if** the logo collides with text, edges, page numbers, or other elements.
- [ ] **PASS if** the logo appears crisp and at sufficient resolution (not pixelated).
- [ ] **FAIL if** the logo appears blurry, pixelated, or has visible compression artifacts.

### Header/Footer
- [ ] **PASS if** page numbers appear on all content slides (5–28 range) in the footer zone.
- [ ] **FAIL if** page numbers are missing, duplicated, or misaligned on content slides.
- [ ] **PASS if** page numbers use Figtree 9pt in text light color (#A0A0A0).
- [ ] **FAIL if** page numbers use wrong font, size, or color.
- [ ] **PASS if** no page numbers appear on cover (1, 2) or closing (29, 30) slides.
- [ ] **FAIL if** page numbers are present on cover or closing slides.

### Image Quality
- [ ] **PASS if** all images appear sharp and non-pixelated at their display size.
- [ ] **FAIL if** any image is visibly pixelated, stretched, or has compression artifacts.
- [ ] **PASS if** all images maintain their original aspect ratio (no stretching/squashing).
- [ ] **FAIL if** any image has been distorted from its original proportions.
- [ ] **PASS if** no stock photo watermarks are visible.
- [ ] **FAIL if** any image shows a stock photo watermark or preview marker.

### Color Contrast
- [ ] **PASS if** all text meets WCAG AA contrast ratios against its background (4.5:1 body, 3:1 large text).
- [ ] **FAIL if** text is placed on backgrounds without sufficient contrast for readability.
- [ ] **PASS if** text on images uses an overlay, shadow, or bounding box to ensure readability.
- [ ] **FAIL if** text is placed directly on photographs without readability treatment.

### Cross-Slide Consistency
- [ ] **PASS if** fonts, sizes, colors, and spacing are consistent for similar elements across all slides.
- [ ] **FAIL if** the same type of element (card, heading, body text) uses different styling on different slides without clear purpose.
- [ ] **PASS if** visual treatments (card borders, shadows, backgrounds) are uniform throughout.
- [ ] **FAIL if** visual style randomly changes between slides indicating copy-paste from different sources.

## 7) Layout Variety & Template Utilization

### Layout Diversity
- [ ] **PASS if** the deck uses at least **5 distinct slide layouts** from the 30-layout template catalog.
- [ ] **WARN if** the deck uses only 3–4 distinct layouts. The deck may feel monotonous and assembled rather than designed.
- [ ] **FAIL if** the deck uses fewer than 3 distinct layouts. A deck built on one or two recycled layouts signals careless assembly.
- [ ] **PASS if** layout choices match slide purpose (e.g., comparison layouts for comparisons, process layouts for sequences, not generic bullet layouts for everything).
- [ ] **FAIL if** a generic bullet layout is used for content that clearly belongs in a comparison, process, grid, or visual storytelling layout.

### Layout-Content Fit
- [ ] **PASS if** each slide uses a layout appropriate for its content type and information density.
- [ ] **FAIL if** a 3-column capability layout is used with only 1 or 2 items populated (wasted layout space).
- [ ] **FAIL if** a single-topic deep-dive layout is overloaded with content that needs a grid or multi-column layout.
- [ ] **PASS if** data-heavy slides use grid, table, or metric layouts rather than forcing numbers into prose paragraphs.
- [ ] **FAIL if** quantitative data or comparisons are presented as running text when a structured layout would be clearer.

### Template Slide Hygiene
- [ ] **PASS if** no unused template placeholder slides remain in the final deck.
- [ ] **FAIL if** blank or sample-text template slides are left in the delivered deck.
- [ ] **PASS if** cover slides use layout 1 or 2, and closing slides use layout 29 or 30.
- [ ] **FAIL if** cover or closing positions use a content layout, or content positions use a cover/closing layout.
- [ ] **PASS if** section dividers (layout 4) are used intentionally to separate major sections, not scattered randomly.
- [ ] **WARN if** a deck over 15 slides has no section dividers. Longer decks benefit from clear section breaks.

### Non-Native Content Checks
- [ ] **PASS if** all diagrams, flowcharts, and architecture visuals are built with native PowerPoint shapes, connectors, and text.
- [ ] **FAIL if** Mermaid diagram renders, Visio exports, or other tool-generated diagram images are pasted in without being rebuilt as native PPTX elements.
- [ ] **PASS if** embedded images serve a clear purpose (product screenshots, photos, evidence) rather than substituting for editable content.
- [ ] **FAIL if** text-heavy content is delivered as an image export instead of native editable text.

---

## Reviewer Verdict Rule

A deck is **brand compliant** only if all critical checks pass:
- Figtree-only typography
- approved color palette only
- no modified logos or protected decorative brand shapes
- correct template size and slide structure
- no baked-in text or screenshot-based charts/tables
- no text overflow, element overlaps, or spacing violations
- consistent sizing, positioning, and visual treatment across slides
- readable color contrast on all text
- crisp, undistorted images
