## Key Lesson Learned

**Client-centric framing is the single biggest lever for IP stress test scores.** The Loop 1 draft was technically clean on IP (zero violations) but scored 27/50 because it was a product description, not a proposal section. The same content, restructured with an institutional opening ("Why Architecture Matters for Institutional Tokenization"), "For institutions..." framings, and benefit-first language, jumped 10 points without adding any new technical content. IP discipline is necessary but not sufficient; the real test is whether sanitized content still reads as a persuasive proposal rather than a sterilized product sheet.

Secondary lesson: Em dashes are still appearing in first drafts from source content. The Loop 1 draft had 8 em dashes copied through from the content sections. Building a pre-write mental filter for dashes would save review time.
