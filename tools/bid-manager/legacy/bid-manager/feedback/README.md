# Feedback Directory

This is where all learning inputs go. Every piece of feedback, every correction, every client reaction feeds back into the bid-writing process to make future bids better.

## Three Categories

### Bid Feedback
What worked, what didn't, reviewer corrections, client responses. This includes:
- Direct comments on specific bid sections ("too technical", "loved the compliance matrix")
- Reviewer edits and the reasoning behind them
- Client follow-up questions (signal of what wasn't clear enough)
- Won/lost outcome notifications with any available evaluator comments

### Meeting Notes
Client calls, internal reviews, demo feedback, anything that reveals what matters to buyers:
- What questions did the client ask? (reveals priorities)
- What concerned them? (reveals objections to address proactively)
- What did they praise? (reveals what to double down on)
- What competitors did they mention? (reveals positioning needs)

### Lessons
Distilled patterns extracted from feedback that should influence all future bids. These live in `lessons.md` and are organized by category. Raw feedback gets logged; lessons are the refined, actionable takeaways.

## How It Works

1. Feedback arrives (any form, verbal, written, meeting notes, outcome notifications)
2. It gets logged in `feedback-log.md` with date, source, type, and summary
3. Actionable lessons are extracted and added to `lessons.md`
4. If the feedback corrects content or reveals new patterns, upstream files get updated too

See `../setup/learning-protocol.md` for the full processing rules.
