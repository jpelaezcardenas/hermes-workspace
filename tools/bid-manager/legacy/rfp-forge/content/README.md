# Content

This directory existed for the old RFP responder workflow.

Those responder-side content libraries are no longer the operating model for RFP Forge.

## New Purpose

RFP Forge now generates buyer-side procurement documents.

Until a new generator-side content system is defined, the subdirectories in this area should be treated as placeholders only. They may later hold:
- procurement boilerplate
- institutional requirement libraries
- jurisdiction-specific clauses
- scoring templates
- appendices and submission instruction patterns

## Current Rule

Do not treat the old content structure as active reusable answer inventory. It has been cleared as part of the overhaul.
