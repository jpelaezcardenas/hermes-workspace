# RFP Structures

Use these templates to match the procurement document to the institution's maturity, sourcing intent, and likely operating style.

Choose the structure that best fits the dossier evidence and procurement context.

---

## Type A: Full RFP

**Best for:** CLM 3-5 institutions running a structured, competitive sourcing process.
**Typical length:** 40-80 pages (excluding appendices).
**Response period:** 4-6 weeks minimum.

### Recommended Structure

1. **Cover page** — institution name/logo placeholder, document title, reference number, issue date, submission deadline
2. **Confidentiality notice** — handling restrictions, NDA requirement, information return/destruction obligations
3. **Table of contents**
4. **Executive summary / background**
   - institutional context and strategic objectives
   - business drivers for this procurement
   - current state and target state
   - scope boundaries and exclusions
5. **Definitions and glossary**
6. **Scope of work**
   - functional scope (asset lifecycle management, issuance, servicing, compliance, settlement, custody, reporting)
   - technical scope (architecture, integration, security, scalability, deployment)
   - operational scope (support, maintenance, training, knowledge transfer)
   - geographic and jurisdictional scope
7. **Functional requirements** (FR-series)
   - asset lifecycle management
   - issuance and origination
   - servicing and corporate actions
   - compliance controls and regulatory reporting
   - settlement and post-trade
   - custody considerations
   - reporting, analytics, and dashboards
8. **Technical requirements** (TR-series)
   - architecture and platform design
   - integration and interoperability
   - security and access management
   - scalability and performance
   - deployment model (cloud, on-premise, hybrid)
   - data management and migration
9. **Compliance and regulatory requirements** (CR-series)
   - jurisdiction-specific regulatory alignment
   - audit and reporting capabilities
   - data privacy and protection
   - record retention and immutability
10. **Security requirements** (SR-series)
    - information security standards (ISO 27001, SOC 2)
    - cryptographic key management
    - penetration testing and vulnerability management
    - business continuity and disaster recovery
11. **Implementation requirements** (IR-series)
    - implementation methodology and timeline
    - project governance and resource plan
    - data migration and transition approach
    - training and knowledge transfer
    - go-live criteria and acceptance testing
12. **Vendor qualification criteria** — mandatory pass/fail gates
    - company profile and financial standing
    - relevant experience and reference clients
    - certifications and accreditations
    - insurance and liability coverage
13. **Evaluation methodology**
    - evaluation gates (administrative → qualification → scored)
    - category weightings with scoring rubric
    - two-envelope or single-envelope approach
    - evaluation panel composition
    - clarification and demonstration process
14. **Commercial requirements** (CM-series)
    - pricing structure and format
    - licensing model
    - SLA expectations and penalty framework
    - payment terms and milestones
    - total cost of ownership requirements
15. **Contractual and legal framework**
    - draft contract terms or term sheet
    - intellectual property provisions
    - data ownership and portability
    - termination and exit provisions
    - liability and indemnification
    - dispute resolution
16. **Timeline and milestones**
    - procurement timeline (issue → Q&A → submission → evaluation → award → contract)
    - implementation timeline expectations
17. **Submission instructions**
    - format requirements (electronic, copies, file types)
    - response structure requirements
    - page/word limits where applicable
    - contact details and communications protocol
    - proposal validity period
18. **Appendices**
    - glossary
    - response templates and forms
    - pricing workbook template
    - reference questionnaire template
    - SLA template
    - data sheets and architecture context diagrams
    - conflict-of-interest declaration form
    - NDA template

### Notes
- Include weighted scoring methodology with published rubric.
- Use detailed requirement numbering across all domains.
- Mandatory requirements should have pass/fail designation.
- Response templates in appendices ensure vendor comparability.

## Type B: RFI (Request for Information)

**Best for:** CLM 1-3 institutions in exploration or market-scanning mode.
**Typical length:** 15-25 pages.
**Response period:** 2-3 weeks.
**Important:** Include explicit statement that the RFI does not commit the institution to any procurement.

### Recommended Structure

1. **Cover page**
2. **Introduction and disclaimer**
   - purpose of the RFI
   - explicit statement: "This RFI is for information-gathering purposes only and does not constitute a commitment to procure"
   - how information will be used
3. **Institutional background**
   - organization overview
   - strategic context and objectives
   - current state of digital asset capabilities
4. **Information requested**
   - company overview and market positioning
   - platform capabilities and architecture overview
   - regulatory compliance alignment
   - deployment options and flexibility
   - implementation approach and typical timelines
   - reference clients in financial services
   - pricing model and commercial structure
   - product roadmap and innovation pipeline
5. **Specific questions** (structured but not as formal as RFP requirements)
   - functional capability questions
   - technical architecture questions
   - compliance and security questions
   - operational and support questions
6. **Response format and guidelines**
   - suggested response structure
   - page/word limits
   - format requirements
7. **Timeline**
   - RFI issue date
   - Q&A deadline (if applicable)
   - response deadline
   - expected next steps and timeline for potential RFP
8. **Contact information**
9. **Appendix: Response template** (optional but recommended)

### Notes
- Use a lighter, information-gathering tone — the institution is learning.
- Focus on qualification, market understanding, and solution discovery.
- Avoid heavy evaluation methodology — but response instructions should still be clear.
- Include clear statement about non-commitment and how information will be handled confidentially.
- Consider including a brief market landscape section to show vendors the institution has done homework.

## Type C: Tender / ITT (Invitation to Tender)

**Best for:** CLM 4-5 institutions procuring against a defined project scope with high commercial rigor.
**Typical length:** 50-120 pages (excluding appendices).
**Response period:** 4-8 weeks.

### Recommended Structure

1. **Invitation to Tender notice** — formal notice with reference number, contracting authority, project title
2. **Instructions to Tenderers**
   - tender process overview
   - communications protocol and contact restrictions
   - site visits or briefing sessions (if applicable)
   - tender submission requirements
   - tender validity period
   - reservation of rights
3. **Background and context**
   - contracting authority description
   - project background and objectives
   - current systems and operations
   - strategic alignment
4. **Scope of work and detailed specifications**
   - comprehensive specification of required deliverables
   - service levels and performance standards
   - acceptance criteria for each deliverable
   - interfaces with existing systems
5. **Technical requirements** (detailed, testable, numbered)
6. **Commercial requirements**
   - pricing schedule (mandatory format)
   - licensing and subscription terms
   - SLA framework with service credits
   - payment schedule and milestones
   - price escalation mechanism
7. **Qualification criteria** (mandatory pass/fail)
   - financial standing thresholds (minimum revenue, solvency ratios)
   - relevant experience (minimum years, minimum reference implementations)
   - certifications (ISO 27001, SOC 2 Type II, etc.)
   - insurance coverage minimums
   - debarment and sanctions checks
8. **Evaluation methodology**
   - multi-gate evaluation:
     - Gate A: Administrative Compliance (Pass/Fail)
     - Gate B: Financial Standing (Pass/Fail)
     - Gate C: Capability Assessment (Pass/Fail)
     - Gate D: Technical Evaluation (Scored)
     - Gate E: Commercial Evaluation (Scored)
   - MEAT (Most Economically Advantageous Tender) assessment
   - two-envelope system: technical evaluated before commercial
   - scoring rubric with definitions
   - evaluation panel composition
9. **Contractual terms and conditions**
   - draft contract (or reference to standard terms)
   - intellectual property
   - data protection and GDPR/equivalent compliance
   - confidentiality
   - termination provisions
   - force majeure
   - liquidated damages
   - dispute resolution
   - insurance requirements
   - parent company guarantee (if applicable)
10. **Submission requirements**
    - envelope/package structure (technical vs commercial)
    - mandatory forms and declarations
    - response format, binding, and labeling requirements
    - submission deadline (date, time, timezone)
    - submission method (portal, email, physical)
11. **Appendices**
    - pricing workbook
    - response templates
    - draft SLA
    - draft contract
    - conflict-of-interest declaration
    - code-of-integrity declaration
    - NDA
    - glossary
    - reference site visit schedule (if applicable)

### Notes
- Pass/fail gates must be unambiguous.
- Commercial and contractual language should be significantly more developed than in an RFI.
- Two-envelope system prevents price from influencing technical evaluation.
- Include liquidated damages framework for SLA breaches.
- Include transition-in and transition-out requirements.

## Type D: Public Sector Tender

**Best for:** Government agencies, central banks, development banks, sovereign entities under formal procurement regulations.
**Typical length:** 60-200 pages.
**Response period:** 6-12 weeks.

### Recommended Structure

1. **Notice of Invitation** — published per regulatory requirement (OJEU, SAM.gov, national gazette, e-procurement portal)
2. **Instructions to Bidders**
   - eligibility criteria
   - bid preparation instructions
   - bid submission requirements (often very prescriptive)
   - bid security / Earnest Money Deposit (EMD)
   - bid validity period
   - right to accept/reject
   - code of integrity / anti-corruption declaration
3. **Procurement context**
   - contracting authority
   - project background
   - applicable procurement regulations (EU Directive 2014/24/EU, UK Procurement Act 2023, World Bank Procurement Framework, national procurement law)
   - funding source (if development bank or donor-funded)
4. **Scope of work** — detailed specification
5. **Technical specifications** — granular, testable requirements
6. **Financial bid format** — mandatory pricing template
7. **Qualification criteria** (pass/fail, per procurement regulations)
   - legal standing
   - economic and financial capacity
   - technical and professional ability
   - conflict-of-interest declarations
   - debarment checks
   - sanctions screening
   - small business / local content requirements (if applicable)
8. **Evaluation methodology**
   - Quality-Cost Based Selection (QCBS) — common for World Bank
   - MEAT assessment — standard for EU
   - multi-gate evaluation with published methodology
   - evaluation committee terms of reference
9. **General and Special Conditions of Contract**
   - payment conditions
   - performance guarantees
   - intellectual property
   - audit rights (common in public sector)
   - transparency and freedom of information obligations
   - social value and sustainability requirements
   - warranty and defect liability
10. **Bid submission forms** (often prescribed templates)
    - bid form
    - price schedule
    - qualification information form
    - JV/consortium agreement (if applicable)
    - performance security format
    - advance payment guarantee format
11. **Appendices**
    - draft contract
    - technical drawings/specifications
    - site information
    - glossary
    - relevant regulatory extracts

### Notes
- Public sector procurement has strict legal requirements — structure must comply with applicable procurement law.
- Advertising, standstill periods, and debriefing obligations apply.
- Special attention to transparency and audit trail.
- Social value, sustainability, and local content criteria may be mandatory.
- World Bank-funded procurement uses specific Standard Procurement Documents (SPDs).

## Type E: Framework Agreement

**Best for:** Institutions with ongoing, multi-use procurement needs across multiple projects or departments.
**Typical length:** 40-60 pages.
**Response period:** 4-6 weeks.

### Recommended Structure

1. **Cover page and invitation**
2. **Framework overview**
   - purpose and scope of the framework
   - duration (typically 3-4 years with extension options)
   - estimated total value range
   - number of suppliers to be appointed (single or multi-supplier)
   - call-off mechanism (mini-competition, direct award, cascading)
3. **Scope of services** — broad capability requirements
4. **Qualification criteria** (to join the framework)
5. **Evaluation methodology** (for framework admission)
6. **Call-off procedures**
   - how specific assignments will be awarded
   - mini-competition rules (if multi-supplier)
   - response times for call-offs
   - escalation and dispute resolution
7. **Commercial framework**
   - rate card structure
   - pricing mechanism for call-offs
   - indexation and price review
8. **Contract terms** (framework agreement + call-off contract template)
9. **Submission requirements**
10. **Appendices** — rate card template, call-off template, performance reporting template

### Notes
- Framework agreements separate qualification from specific project awards.
- Multi-supplier frameworks use mini-competitions for individual assignments.
- Common in EU public sector procurement and large enterprises.

## Type F: Pre-Qualification Questionnaire (PQQ) / Selection Questionnaire (SQ)

**Best for:** Vendor shortlisting before issuing the main ITT/RFP, especially in restricted procedures.
**Typical length:** 10-20 pages.
**Response period:** 2-3 weeks.

### Recommended Structure

1. **Introduction and process overview**
2. **Company information**
3. **Grounds for mandatory exclusion** (pass/fail)
   - criminal convictions
   - failure to pay taxes
   - bankruptcy/insolvency
4. **Grounds for discretionary exclusion** (pass/fail)
   - professional misconduct
   - conflict of interest
   - significant deficiencies in prior contracts
5. **Economic and financial standing**
   - annual turnover thresholds
   - financial ratios
   - insurance coverage
6. **Technical and professional ability**
   - relevant experience and references
   - certifications and accreditations
   - key personnel qualifications
   - quality management systems
7. **Additional information** (if needed)
8. **Declaration and submission instructions**

### Notes
- PQQ/SQ is used in restricted procedures to create a shortlist before issuing the full ITT.
- In UK public sector, the standardized Selection Questionnaire is prescribed.
- Keep proportionate — PQQ requirements should not exceed what's necessary for the procurement value.

## Selection Guidance

### By Institution Maturity
| CLM Level | Primary Type | Secondary Type |
|---|---|---|
| CLM 1-2 | Type B (RFI) | — |
| CLM 2-3 | Type B (RFI) or Type A (Light RFP) | Type F (PQQ) if shortlisting |
| CLM 3-4 | Type A (Full RFP) | Type F (PQQ) → Type A |
| CLM 4-5 | Type C (Tender/ITT) | Type F (PQQ) → Type C |

### By Procurement Context
| Context | Recommended Approach |
|---|---|
| First-time exploration | Type B (RFI) |
| Known market, clear requirements | Type A (Full RFP) |
| Defined project, high commercial rigor | Type C (Tender/ITT) |
| Public sector / regulated procurement | Type D (Public Sector Tender) |
| Ongoing multi-year need | Type E (Framework Agreement) |
| Large field, need to shortlist | Type F (PQQ) → Type A or C |

### Multi-Stage Procurement
For complex, high-value procurements, consider multi-stage approaches:

1. **RFI → RFP**: RFI to scan market, shortlist 5-8 vendors, then full RFP to 3-5
2. **PQQ → ITT**: Pre-qualification to shortlist, then detailed tender to qualified vendors
3. **RFI → RFP → BAFOs**: Full three-stage for sovereign or large bank procurements
4. **Competitive Dialogue**: For innovation procurement where solution approach is undefined — structured dialogue rounds before final tender submission

When unsure, prefer the simpler structure that still feels credible for the institution's maturity. Over-engineering the procurement process wastes vendor effort and institutional evaluation resources.
