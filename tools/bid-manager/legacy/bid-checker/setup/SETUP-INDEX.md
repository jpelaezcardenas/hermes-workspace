# Setup Index

This file is the navigation map for the Bid Checker knowledge base.
Read it after `SOUL.md` so you know which setup file to use for which judgement.

- `scoring-rubric.md` -- Defines the 10 scoring dimensions, anchors, and what each score level means. Includes expanded dimensions for factual accuracy, positioning alignment, audience appropriateness, and competitive differentiation.
- `ip-checklist.md` -- Lists confidentiality, internal-term, and metadata leak checks that can block submission.
- `review-template.md` -- Defines the required structure and sections for every output review, including the expanded review format.
- `writing-standards.md` -- Explains how to judge proposal prose, paragraph quality, format choice, voice consistency, and quantitative writing benchmarks.
- `reading-psychology.md` -- Explains how evaluators actually scan, triage, remember, and compare proposal documents under pressure.
- `persuasion-framework.md` -- Explains how to test argument quality, trust formation, evidence logic, and persuasive structure.
- `structure-patterns.md` -- Explains hierarchy, information architecture, section sequencing, TOC quality, and retrieval logic.
- `evaluator-personas.md` -- Models the main evaluator archetypes and the defects each persona notices first.
- `defect-taxonomy.md` -- Classifies common proposal defects, severity levels, detection methods, evaluator impact, and fix patterns.
- `dalp-claim-verification.md` -- DALP capability verification checklist, common overclaims, content guardrails, competitive positioning rules, leadership title verification, and DOCX formatting exclusion rule.
- `tone-assessment.md` -- Criteria for evaluating proposal tone: what makes good tone vs bad, with specific indicators and scoring guidance.
- `common-defect-patterns.md` -- Documented frequent issues seen across proposals, with detection methods and fix patterns.
- `executive-summary-criteria.md` -- Specific criteria for assessing executive summary quality beyond the general rubric.
