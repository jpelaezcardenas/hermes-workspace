# Usage examples

Practical ways to use the stock image library in DALP-heavy decks, proposals, and enterprise pitch material.

## 1. DALP overview / platform story
Use when the slide is about lifecycle orchestration, composability, operating control, or platform-wide coordination.

- `dalp/dalp-01.jpg` — conceptual lifecycle overview / control plane
- `dalp/dalp-02.jpg` — orchestration / modular architecture / workflow automation
- `software/software-04.jpg` — real-world enterprise platform / operator environment
- `software/software-05.jpg` — engineering depth / implementation credibility

## 2. Digital asset narrative
Use when the slide is about tokenized assets, registries, market activity, operator workflows, or asset servicing.

- `digital-assets/digital-assets-04.jpg` — crypto / digital-asset monitoring in a business setting
- `digital-assets/digital-assets-05.jpg` — analyst / operator framing for tokenized products
- `digital-assets/digital-assets-06.jpg` — market-data note-taking / portfolio-review context
- `digital-assets/digital-assets-01.jpg` to `digital-assets-03.jpg` — conceptual support when you want less literal photography

## 3. Compliance / policy / controls
Use when the slide is about KYC, AML, approval workflows, governance, or regulatory operating model.

- `compliance/compliance-04.jpg` — regulated review workflow / documents + laptops
- `compliance/compliance-05.jpg` — policy discussion / working session / governance committee feel
- `compliance/compliance-06.jpg` — professional document review without cheesy stock theatrics
- `custody/custody-01.jpg` — controlled access / security posture / permissioned environment

## 4. Custody / safekeeping / security
Use when the message is institutional trust, controlled environments, or infrastructure-grade protection.

- `custody/custody-01.jpg` — access control / secure entry / controlled authorization
- `custody/custody-02.jpg` — institutional infrastructure / secure environment / resilience
- `custody/custody-03.jpg` — operational custody team / infrastructure ops

## 5. Settlement / cash movement / payment rails
Use when the slide is about transaction rails, settlement modernization, cash leg orchestration, or payment-side movement.

- `settlement/settlement-01.jpg` — payment initiation / transaction execution
- `settlement/settlement-02.jpg` — payment terminal next to laptop for fintech workflow framing
- `settlement/settlement-03.jpg` — contactless transaction / cashless execution

## 6. Fintech / market infrastructure / analytics
Use when the slide is about capital markets, trading operations, analytics, market-data workflows, or modernization of financial infrastructure.

- `fintech/fintech-01.jpg` — multi-monitor market-data / trading-desk energy

## 7. Software / integration / enterprise delivery
Use when speaking to CTOs, platform owners, architects, or implementation teams.

- `software/software-04.jpg` — operator at multi-monitor setup
- `software/software-05.jpg` — developer / code / implementation narrative
- `software/software-06.jpg` — enterprise software / project management / delivery coordination
- `technology/tech-01.jpg` to `tech-03.jpg` — generic infra support where needed

## 8. Slide pairings that work well

### Pair stock image + DALP screenshot
- Cover/section image: `custody/custody-02.jpg`
- Product proof: a DALP admin / policy / asset-management screenshot

### Pair stock image + architecture diagram
- Cover/section image: `fintech/fintech-01.jpg`
- Diagram slide: market-infrastructure architecture, issuance-to-settlement flow, or ecosystem operating model

### Pair stock image + executive messaging
- Cover/section image: `digital-assets/digital-assets-05.jpg`
- Content: market opportunity, institutional readiness, why now

### Pair stock image + compliance narrative
- Cover/section image: `compliance/compliance-04.jpg`
- Content: rules, approvals, policy templates, role-based controls

## 9. Quick selection rules

### If the slide says...
- **“platform” / “integration” / “implementation” / “API”** → start with `software/`
- **“digital assets” / “operators” / “holders” / “asset servicing”** → start with `digital-assets/`
- **“compliance” / “controls” / “governance” / “policy”** → start with `compliance/`
- **“custody” / “security” / “institutional trust”** → start with `custody/`
- **“settlement” / “payments” / “cash movement”** → start with `settlement/`
- **“market infrastructure” / “trading” / “analytics” / “fintech”** → start with `fintech/`
- **“tokenization lifecycle” / “issuance” / “fractionalization”** → start with `tokenization/`
- **Need something cleaner and less literal** → start with `dalp/` or `abstract/`

## 10. Anti-patterns

- Don’t use a generic business-meeting photo for a custody or compliance slide if a sharper custody/compliance image exists.
- Don’t use any of these images as proof of DALP product functionality. Use screenshots, architecture diagrams, and verified product visuals for that.
- Don’t overuse trading-screen photos across the same deck. One or two is strong; six makes the deck look like a retail broker ad.
- Don’t place dense white text on bright uncropped photos without an overlay. That’s not the image’s fault. That’s bad slide hygiene.
