# Delivery Standards

## Before delivery
- confirm the final document type still matches the request
- ensure DOCX and markdown source are both saved in `output/`
- open the DOCX in Word and check first-page impression, headings, tables, figures, and page boundaries

## Filenames
Prefer descriptive filenames such as:
- `executive-memo-tokenization-readiness-2026-03-15.docx`
- `meeting-summary-partner-enablement-2026-03-15.docx`
- `research-summary-market-signals-q1-2026.docx`

## Delivery note guidance
When handing off, mention:
- what the document is
- who it is for if relevant
- any caveats, assumptions, or omitted confidential detail

## Retention rule
The markdown source stays with the DOCX so the document can be revised safely later.
