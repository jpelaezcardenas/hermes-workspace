# Audience Adaptation

## Rule
The document should adapt to the reader's job, not force every reader through the same structure.

## Executive audience
Lead with:
- outcome
- recommendation
- risk
- decision needed

Use:
- short sections
- tight bullets
- bottom-line-first summaries
- a visible recommendation or ask

Avoid:
- implementation weeds in the opening
- long historical setup
- large reference tables in the main body

## Operational audience
Lead with:
- process
- ownership
- actions
- timing

Use:
- action tables
- clear owners
- dates, dependencies, and follow-ups
- explicit next steps

Avoid:
- vague action items
- strategy language with no operating detail

## Technical audience
Lead with:
- structure
- dependencies
- decisions
- constraints

Use:
- precise terms
- assumptions and caveats
- evidence and architecture logic
- appendices for deeper detail

Avoid:
- marketing language
- hand-wavy recommendations

## Mixed audience
Use a layered structure:
1. purpose and takeaway
2. implications for decision-makers
3. operating or technical detail
4. appendix if needed

## External client audience
Use:
- clean context-setting
- plain language
- careful handling of assumptions
- client-safe wording

Avoid:
- internal shorthand
- drafty notes language
- unverified product claims

## Internal leadership audience
Use:
- direct trade-offs
- explicit recommendation
- crisp risks and mitigations
- no ceremony
