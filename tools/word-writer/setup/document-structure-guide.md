# Document Structure Guide

## Rule
The structure must match the reader's job. If the reader is deciding, front-load the decision. If the reader is learning, front-load context and findings. If the reader is acting, front-load actions and deadlines.

## Default structures by format

### Report
Title -> Executive Summary -> Context -> Findings -> Recommendations -> Next Steps -> Appendix

### Executive Memo
Title -> Decision / Ask -> Why It Matters -> Key Points -> Risks -> Recommendation

### Structured Briefing
Title -> Situation -> What Changed -> What Matters -> Implications -> Recommended Response

### Q&A Pack
Title -> Context -> Questions and Answers -> Open Issues -> Appendix

### Meeting Summary
Title -> Meeting Details -> Key Decisions -> Discussion Summary -> Actions -> Risks / Follow-ups

### Status Update
Title -> Current Status -> Changes Since Last Update -> Risks / Issues -> Decisions Needed -> Next Actions

### Research Summary
Title -> Research Question -> Scope / Method -> Findings -> Implications -> Recommended Next Steps

### Operating Note / Guidance Memo
Title -> Purpose -> Rule or Guidance -> Exceptions -> Actions / Owners -> Reference Notes

## Section design rules
- Start with the answer or purpose, not a throat-clearing preamble.
- Keep main sections mutually distinct.
- Use appendices for supporting detail, not for essential takeaways.
- If the document is under two pages, keep the structure compact.
- If the document exceeds six pages, consider stronger sectional grouping and a TOC.
