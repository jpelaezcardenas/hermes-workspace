# Caption and Figure Rules

## Core rules
- Add captions when the image or diagram carries evidence, explanation, or a specific claim.
- Keep captions short and factual.
- Place the figure after the paragraph that introduces it.
- Do not dump images without context.
- If an image adds no informational value, remove it.

## Caption pattern
Use one of these patterns:
- `Figure 1. Current workflow from intake to delivery.`
- `Figure 2. Architecture view of the document generation flow.`
- `Figure 3. Example status dashboard used in weekly review.`

## Figure usage standards
- One figure, one idea.
- Prefer clean screenshots over cluttered full-screen captures.
- Crop irrelevant browser chrome or dead space when possible.
- Do not stack many figures back to back without commentary.
- If a figure is decorative rather than informative, cut it.

## Lead-in rule
Before a figure, include one sentence that tells the reader why it matters.

## After-figure rule
If the meaning is not obvious on sight, add one sentence of interpretation after the figure.
