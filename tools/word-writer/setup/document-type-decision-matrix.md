# Document Type Decision Matrix

## Fast chooser

| Need | Best document type | Default TOC | Notes |
|---|---|---:|---|
| A decision or recommendation | Executive Memo | No | Lead with ask and recommendation |
| Multi-section analysis with evidence | Report | Yes | Use appendix for supporting detail |
| Fast situational update | Structured Briefing | No | Keep implications visible |
| Anticipated stakeholder questions | Q&A Pack | Maybe | Use TOC only if long |
| Record of a meeting and next steps | Meeting Summary | No | Actions need owners and dates |
| Repeating program or delivery update | Status Update | No | Use tables for owners and risks |
| Synthesis of source material | Research Summary | Maybe | Be explicit about scope and confidence |
| Rules, process guidance, or operating instructions | Operating Note / Guidance Memo | No | Call out exceptions and owners clearly |
| Prompt/response collection | Questionnaire / Response Pack | Maybe | Keep answer blocks concise |

## Boundary checks
- If the document needs competitive persuasion to win work, that is not Word Writer.
- If the document is the procurement instrument itself, that is not Word Writer.
- If the document is meant for journalists or media, that is not Word Writer.

## Tie-breaker rule
When in doubt, choose the lighter format first. Most bad Word documents are overbuilt, not underbuilt.
