# Table Style Rules

1. Use tables for comparison, status, schedules, owners, risks, and structured facts.
2. Do not use tables for dense prose.
3. Header row should name the decision dimension clearly.
4. Prefer 3 to 6 columns.
5. Prefer 3 to 8 body rows.
6. Split oversized tables instead of shrinking readability.
7. Keep cell content short.
8. Use parallel phrasing row to row.
9. If a table needs interpretation, add one sentence before or after it.
10. Final check: would this still be readable when opened in desktop Word at 100% zoom?

## Better table patterns
- owner / action / due date
- option / pros / cons / recommendation
- risk / impact / mitigation / owner
- question / answer / source confidence

## Bad table patterns
- paragraph-long cells in every column
- too many narrow columns
- mixing unrelated dimensions in one table
- hiding conclusions inside a table with no surrounding narrative
