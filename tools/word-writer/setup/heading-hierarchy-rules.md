# Heading Hierarchy Rules

1. H1 is the document title only.
2. The subtitle or date sits as plain text below the H1, not as another H1.
3. H2 defines the main sections.
4. H3 supports H2 sections.
5. H4 is rare and should be used only when it improves scanning.
6. Do not jump from H2 to H4.
7. If a section has one child heading, it probably needs restructuring.
8. Headings should be informative, not cute.
9. Heading text should tell the reader what the section does, not just label a topic vaguely.
10. If the main body starts with an H2, the H2 should not repeat the title.

## Good heading examples
- `## Key Decisions`
- `## Risks and Mitigations`
- `### Dependency on legal review`

## Weak heading examples
- `## Overview`
- `## Thoughts`
- `### More`
