# Final QA Checklist

## Framing
- [ ] document type is correct for the request
- [ ] correct agent boundary observed
- [ ] audience is obvious from the opening
- [ ] recommendation, ask, or takeaway is clear
- [ ] confidentiality level matches the audience

## Pre-conversion draft check
- [ ] title and subtitle are clean
- [ ] heading hierarchy is logical
- [ ] TOC decision is intentional, not defaulted blindly
- [ ] sections are in the right order for the reader's job
- [ ] tables fit the message rather than replacing explanation
- [ ] diagrams and images are necessary, not ornamental

## Post-conversion DOCX check
- [ ] no obvious blank pages
- [ ] no accidental double page breaks
- [ ] TOC behavior matches the chosen setting
- [ ] tables fit and scan well in Word
- [ ] diagrams and images are centered and legible
- [ ] captions are present when figures carry meaning
- [ ] numbering, bullets, and heading styling are consistent
- [ ] first-page impression looks deliberate, not auto-generated

## Writing
- [ ] opening tells the reader what this is
- [ ] summary is actually useful
- [ ] no em dashes
- [ ] no AI markers
- [ ] no unsupported claims
- [ ] tone matches audience
- [ ] no filler phrasing or awkward hedging
- [ ] actions, owners, and dates are explicit where needed

## Delivery
- [ ] markdown saved in output/
- [ ] DOCX generated in output/
- [ ] final DOCX reviewed after conversion
- [ ] filenames are clear and client-safe
- [ ] delivery notes mention any caveats or assumptions when needed
