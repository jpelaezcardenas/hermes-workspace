# Customer Reference Handling Operating Note
2026-03-15 | Scope: External-facing enablement materials | Owner: Solutions Operations

## Purpose
This note sets the minimum handling rule for customer references used in partner packs, research summaries, and reusable sales support documents.

## Rule or Guidance
- Use named customer references only when written approval is current and traceable
- If approval is pending, use an approved pseudonym or remove the example entirely
- Keep the source of approval in the working file or linked request material
- Treat screenshots that expose customer names, URLs, or account data as reference material and review them under the same rule

## Exceptions
- Legal may approve a narrow one-time use with an expiry date or channel restriction
- Internal-only documents may use named references when access is limited to the approved working group

## Actions / Owners
| Action | Owner | Due |
|---|---|---|
| Check reference approval status before final packaging | Document owner | Before draft sign-off |
| Escalate unresolved approvals | Solutions Operations | Same business day |
| Confirm screenshot scrub or replacement | Product marketing | Before DOCX conversion |

## Reference Notes
Use this note with the final QA checklist and the delivery standard for any document that could leave the immediate project team.
