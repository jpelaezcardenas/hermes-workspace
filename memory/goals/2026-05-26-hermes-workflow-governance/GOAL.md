# Goal: Hermes Workflow Governance v1

Oberziel: Chris' Hermes-Workflows sollen zuverlaessiger, kleinteiliger und besser kontrollierbar werden. Grosse Vorhaben sollen weiter moeglich bleiben, aber als Slice-Kette mit CEO-Watchdog, Memory-Regeln und sichtbaren Artefakten.

## Erfolgskriterien

- Eine globale Kurzregel steht in Hermes Built-in Memory.
- Eine ausfuehrliche Workflow-Regel steht unter `~/.hermes/memory/workflows`.
- Ein wiederverwendbarer CEO-Execute-Goal-Prompt steht unter `~/.hermes/memory/prompts`.
- Schulwerkstatt-Projektmemory kennt diese Regeln.
- Ein Hermes-Task prueft die Integration und meldet, ob die Regeln fuer zukuenftige Tasks abrufbar sind.

## Nicht-Ziele

- Keine tiefen Codeaenderungen an Hermes selbst.
- Keine Modell-/Provider-Umstellung.
- Keine neuen externen Tools.
- Kein Push/Commit/Deploy.
