Each of the five ideas must use a different lens. Rotate and name the lens.

```text

```

Lens examples:

- `Unbundle`: Pull one valuable piece out of a broad system.
- `Productize`: Turn repeated help into a named package.
- `Workflow Compression`: Cut a recurring 60-minute task to 10 minutes.
- `Audience Shift`: Repackage a teacher tool for VDS, parents, trainees, or other teachers.
- `Material First`: Start with PDF, card, checklist or one-page guide.
- `Boring Business`: Useful, paid, unglamorous, low-risk service.

## 4. Chris-Fit Filter

An idea passes only if at least six of eight are true:

```text
1. Uses Chris' real school/GE/AI context.
2. Can be tested in 7 days.
3. First test costs 0-50 EUR.
4. Does not require a shop, platform, company or public launch.
5. Does not require real student data.
6. Has a plausible buyer/user beyond Chris.
7. Uses existing Hermes evidence or materials.
8. Has a clear stop/kill condition.
```

If fewer than six are true, mark the idea:

```text
Label: Creative but not Chris-ready
```

Such an idea may appear only as the Wildcard.

## 5. Scoring Model

Each idea gets these scores:

| Metric | Meaning |
|---|---|
| Chris-Fit | Does this fit his current life and strengths? |
| Creativity | Is this meaningfully non-obvious? |
| 7-Day Testability | Can it be tested this week? |
| Money Potential | Could it later create savings, income or leverage? |
| Start Cost | Higher score means lower cost. |
| Energy / Fun | Would Chris likely want to touch this? |
| Risk Safety | Higher score means safer. |
| Evidence Strength | Is it anchored in Hermes/local evidence? |

CEO Score is not a simple average. It is a judgment:

```text
Would a smart, tired Chris thank Hermes for making this the one action today?
```

## 7. Anchoring Loops

### Daily Loop: Diverge -> Score -> Select -> Act

```text
1. Read anchors.
2. Generate five directions.
3. Score each.
4. Pick one winner.
5. Give one safe next action.
6. Store report.
7. Send Telegram summary.
```

### Weekly Loop: Winners -> Pattern -> Test

At the end of each week, a future weekly job may compare seven winners:

```text
Which idea repeated?
Which idea had strongest Chris-fit?
Which idea was actually touched?
Which one deserves a real 7-day product test?
Which should be killed?
```

### Monthly Loop: Memory -> Rules -> Better Ideas

Once a month, Memory Curator may promote only stable patterns:

```text
Keep: proven Chris-fit rules.
Drop: repeated noise.
Adjust: scoring weights.
Never store: sensitive data or raw private details.
```

## 8. Kill Criteria

An idea must be killed or parked if:

- It needs public launch before private test.
- It needs personal student data.
- It needs a paid provider before value is proven.
- It is basically a trading idea.
- It requires more than 2 hours for first test.
- It creates another dashboard instead of a user outcome.
- It repeats a recent idea without a new angle.
- It feels clever but has no user, problem or test.
