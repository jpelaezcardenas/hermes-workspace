# Execute Plan: Hermes Classroom-Ready Pilotpaket v1

## Ziel

Ein einziges kleines GE-Pilotpaket erstellen, das morgen im Unterricht nutzbar waere.

## Arbeitsmodus

- CEO-gefuehrt.
- Quellenbasiert.
- Kleinster nutzbarer Slice.
- Keine neue Technik.

## Phase 1: Quellen einsammeln

Lies nur die naheliegenden lokalen Quellen:

1. `reports/hermes-control/hermes-control-2026-06-01.md`
2. `reports/decision-inbox/decision-inbox-2026-06-01.md`
3. `reports/hermes-control/hermes-memory-curator-2026-06.md`
4. GE-Lernwerkstatt Wochenplaene und Material-/Testpilot-Reports, falls vorhanden.
5. LeseWerk Produktklarheit nur als Kontext, nicht als Hauptquelle.

Output:

- Eine Quellenliste mit maximal 8 Eintraegen.
- Eine klare Entscheidung: GE-Minikiste, Mengen-legen oder LeseWerk-Materialkorb.

CEO-Kontrolle:

- Wenn mehr als ein Pilot entsteht, stoppen und auf einen reduzieren.

## Phase 2: Pilotpaket entwerfen

Schreibe das Paket in einfacher Unterrichtssprache:

1. Titel
2. Ziel in einem Satz
3. Dauer: 20-30 Minuten
4. Material: maximal 7 Dinge
5. Kind-Ablauf: maximal 5 Schritte
6. Lehrkraft-Ablauf: maximal 5 Schritte
7. Drei UK-/Hilfesignal-Karten
8. Beobachtungsbogen mit maximal 5 Fragen
9. Was nicht bewertet wird
10. Naechster Mini-Slice

CEO-Kontrolle:

- Keine Diagnose-Sprache.
- Keine Defizit-Sprache.
- Keine Punkte, Timer, Ranking oder Noten.

## Phase 3: Praxis-Check

Pruefe das Paket gegen vier Fragen:

1. Kann eine Lehrkraft es in unter 5 Minuten lesen?
2. Ist das Material realistisch auf einem Tisch?
3. Kann ein Kind den ersten Schritt ohne lange Erklaerung beginnen?
4. Fuehrt die Beobachtung zu genau einer naechsten Verbesserung?

Output:

- Kurzer Abschnitt `Praxis-Check`.
- Ampel: Green, Yellow oder Red.

CEO-Kontrolle:

- Bei Yellow: nur eine konkrete Nachbesserung.
- Bei Red: Paket nicht ausgeben, sondern kleiner schneiden.
