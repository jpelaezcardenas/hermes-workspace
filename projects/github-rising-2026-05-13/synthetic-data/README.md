# Synthetische Testdaten

Dieser Ordner ist für künstliche, öffentliche und unkritische Testdaten vorgesehen.

Erlaubt:
- kurze selbst geschriebene Markdown-/Textdateien
- synthetische HTML-Seiten
- Beispielcode ohne interne Logik oder Geheimnisse
- Dummy-Prompts ohne Personenbezug
- öffentliche Demo-Dokumente, wenn Lizenz/ToS passen

Nicht erlaubt:
- Schülerdaten, Namen, Noten, Förderpläne, Elternkontakte
- private Dokumente, E-Mails, Chat-Historien, Kalenderdaten
- echte Credentials, Tokens, API-Keys, Cookies
- interne Projektquellen oder produktive Hermes-Konfigurationen

Empfohlene Dateien für spätere Tests:
- `demo-research-notes.md`
- `demo-page-1.html`
- `demo-page-2.html`
- `demo-react-app-spec.md`
- `demo-prompts.txt`

Hinweis: Diese Dateien wurden bewusst noch nicht mit realen Inhalten befüllt. Vor einem Test nur synthetische Inhalte ergänzen.
