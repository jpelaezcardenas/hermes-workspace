# QA-Paket für jede Slice

Diese Vorlage wird nach jedem abgeschlossenen Slice ausgefüllt.

## 1. Kurzreport

- Slice:
- Datum:
- Bearbeiter:
- Ziel des Slices:
- Was wurde geändert?
- Was wurde geprüft?
- Ergebnis:
- Offene Punkte / Risiken:
- Nächster kleinster Schritt:
- Datei/Link zum Beleg:

## 2. Linkcheck-Checkliste

Vor dem Abschluss jedes Slices prüfen:

- [ ] Alle neuen oder angepassten Links wurden geöffnet.
- [ ] Keine 404- oder Fehlerseite beobachtet.
- [ ] Interne Sprünge funktionieren.
- [ ] Externe Links sind absichtlich gesetzt und fachlich sinnvoll.
- [ ] Keine sensiblen Daten in URL, Query oder Titel.
- [ ] Wenn ein Link auf eine neue Seite führt: Zielseite kurz gegengeprüft.
- [ ] Falls ein Download-Link neu ist: Datei lässt sich öffnen und ist plausibel.

## 3. Mobile-QA-Checkliste

- [ ] Auf schmalem Layout geprüft.
- [ ] Inhalte bleiben lesbar ohne horizontales Scrollen.
- [ ] Wichtige Buttons sind erreichbar und nicht überdeckt.
- [ ] Abstände und Schriftgrößen sind brauchbar.
- [ ] Keine Elemente ragen außerhalb des Viewports.
- [ ] Menüs und Dialoge funktionieren auch mobil.
- [ ] Start- und Hauptaktion sind sofort erkennbar.
- [ ] Lange Texte brechen sauber um.

## 4. Handoff-Checkliste

Vor Übergabe an den nächsten Slice oder an den Menschen:

- [ ] Es gibt genau einen aktiven Implementierungs-Slice.
- [ ] Keine echten Schülerdaten, Elterndaten oder anderen identifizierbaren Personendaten verarbeitet.
- [ ] Keine Diagnosen, Verdachtsdiagnosen oder medizinischen Aussagen behauptet.
- [ ] Alle Beispiele sind anonymisiert oder synthetisch.
- [ ] Änderungen sind knapp dokumentiert.
- [ ] Risiken und Unsicherheiten sind klar benannt.
- [ ] Der nächste Schritt ist klein, klar und machbar.
- [ ] Relevante Dateien oder Belege sind verlinkt.

## 5. CEO-Regeln, die immer gelten

- Keine realen Schülerdaten.
- Keine Namen, Geburtsdaten, Adressen, Diagnosen oder Familieninfos.
- Keine Diagnose- oder Heilversprechen.
- Keine unnötige Datensammlung.
- Nur eine aktive Implementierungs-Slice gleichzeitig.
- Wenn eine Information sensibel sein könnte: anonymisieren oder weglassen.
- Wenn Unsicherheit bleibt: markieren statt raten.

## 6. Ausfüllhinweis für Worker

Der Worker soll dieses Paket direkt nach jedem Slice kurz ausfüllen. Es soll nicht perfekt sein, sondern vollständig genug, damit der nächste Schritt sicher entschieden werden kann.
