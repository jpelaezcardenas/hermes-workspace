# Hermes Cockpit Reference Verification

**Verified at:** 2026-05-17T06:23:16Z
**Scope:** Source pitch reference claims for `joeynyc/hermes-hud` and `m0at/hermes-lite`. This check inspected cloned repository files, not just `git ls-remote` existence.
**Rule:** Repository existence is not the same as relevance, and relevance is not the same as every pitch detail being supported.

## Summary

- `joeynyc/hermes-hud` exists and is relevant to Hermes-agent introspection because it reads `~/.hermes` data and renders a terminal dashboard. However, the pitch overstates it as a **Ratatui** reference and as **definitive**; the inspected code is a Python/Textual app, not a Rust/Ratatui app.
- `m0at/hermes-lite` exists and is relevant to Rust/Ratatui multi-pane agent UI patterns, subprocess-backed panes, `@mentions`, broadcast, and per-pane model/session state. However, the pitch wording should avoid implying it is a Cockpit/flight-recorder implementation or a vetted production reference.

## Recommended corrected reference wording

> Reference examples worth inspecting: `joeynyc/hermes-hud` is a Python/Textual Hermes HUD that reads local `~/.hermes` state, sessions, memory/skills/config, processes, and tmux context. `m0at/hermes-lite` is a Rust/Ratatui local coding-agent prototype with multi-pane agent subprocesses, per-pane model/session state, `@agent` routing, pull-back, and broadcast. These are relevant examples, not definitive Hermes Cockpit architecture, and neither should be cited as production-quality or as a complete Kanban-native flight recorder without further verification.
