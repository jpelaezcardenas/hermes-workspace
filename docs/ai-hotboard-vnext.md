# AI 热点看板 vNext

更新时间：2026-04-17
目标：把当前 Hermes Workspace `/dashboard` 改造成可直接展示的 AI 热点看板页面，视觉风格尽量贴近参考图（卡兹克版本），但产品逻辑升级为企业内部情报工作台。

---

## 1. 产品定位

不是传统 BI，也不是资讯站。

定位为：**AI 信号采集 → 事件聚合 → 策略判断 → 推荐动作** 的情报工作台。

核心使用者：
- jc老大 / 管理层：看今日最值得关注的 AI 信号
- 内容/投放/运营：看哪些事件可转动作
- 爱马仕 / 小J / agent：看系统抓到了什么、该继续跟什么

---

## 2. 必须保留的参考模板骨架

从参考图吸收的有效结构：

1. 左侧工作台导航
2. 中间事件卡片流
3. 每条卡片都有推荐理由
4. 多源聚合（另有 N 个源报道）
5. 标签系统
6. 视觉上偏暗色、信息密度高、卡片边界明确

不照搬的部分：

1. 纯资讯社区口吻
2. 只有热度分没有业务评分拆解
3. 过度概念化栏目命名
4. 策略层停留在“观点”，没有动作闭环

---

## 3. 信息架构图

### 3.1 一级结构

- 顶部：全局状态栏 + 时间范围 + 刷新状态
- 左栏：频道导航 / 筛选器 / 系统入口
- 中栏：AI 事件主流（事件卡列表）
- 右栏：选中事件详情 / 信源 / 推荐动作 / 相似事件

---

### 3.2 左栏 IA

#### A. 内容视图
- 今日精选
- 全部 AI 动态
- AI 视频
- 模型发布
- Agent / Workflow
- 低粉爆文
- 白名单信源
- 收藏

#### B. 运营视图
- 待验证
- 待跟进
- 已转动作
- 信源提报
- 策略规则
- 系统状态

#### C. 快速筛选
- 平台：X / HN / Blog / Product / GitHub / YouTube
- 标签：模型 / Agent / 视频 / 图像 / 编码 / 开源 / 工作流
- 业务：短视频 / 投放 / 研发 / 运营 / 品牌

---

### 3.3 顶部总览 IA

建议放 6 个 overview tiles：

1. 今日新增事件数
2. 高价值事件数
3. AI 视频相关数
4. 多源共振事件数
5. 可转动作事件数
6. 白名单命中率

辅助信息：
- last sync time
- 当前时间范围（24h / 3d / 7d）
- 数据状态（Live / Delayed）

---

### 3.4 中央事件流 IA

每个事件卡必须是“事件对象”，不是单条来源。

#### 卡片结构

1. 顶部元信息
   - 时间
   - 热度总分
   - 信源数
   - 状态（精选 / 新增 / 共振 / 待验证）

2. 主体信息
   - 事件标题
   - 3~5 行摘要
   - 一句“为什么重要”

3. 标签区
   - 内容标签：模型发布 / Agent / 视频 / 编码 / 开源
   - 业务标签：可转短视频 / 值得测试 / 值得投放 / 值得老板关注

4. 推荐动作区
   - 观察
   - 测试
   - 转内容
   - 推送给投放组
   - 纳入白名单持续跟踪

5. 多源区
   - 主信源
   - 另有 N 个源也报道了此事件

---

### 3.5 右侧详情区 IA

点选某条事件后展示：

1. 事件标题 + 总分
2. 评分拆解
   - 热度
   - 新颖度
   - 可执行性
   - 业务相关性
   - 持续跟踪价值
3. 原始信源列表
4. 关键词 / 标签
5. 相似事件
6. 推荐动作清单
7. 适用团队
   - 内容
   - 投放
   - 运营
   - 研发

## 5. 视觉规范建议

### 5.1 风格目标
- 暗色背景
- 蓝紫 + 青绿作为主强调色
- 卡片玻璃感轻微，边框明确
- 避免科技大屏那种土味发光
- 强调“编辑感 + 决策感”而非“酷炫图表感”

### 5.2 组件气质
- overview tile：小而锐利，数字醒目
- 事件卡：信息密度高，但结构稳定
- tag：短胶囊样式
- score：右上角亮色 badge
- 右栏：细分 section，多块堆叠

### 5.3 阅读动线
1. 顶部扫全局态势
2. 左边定频道
3. 中间快速扫事件
4. 右边深挖详情
