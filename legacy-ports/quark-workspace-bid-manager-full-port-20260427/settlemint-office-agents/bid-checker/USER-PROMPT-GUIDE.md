# Bid Checker — Prompt Guide

## What this agent does

Bid Checker reviews proposal drafts, RFI responses, and bid documents before submission. It behaves like a senior evaluator on a regulated financial institution's procurement committee — scoring the document across 10 dimensions, checking for IP leaks, pressure-testing the argument through multiple evaluator personas (technical, business, risk/compliance, procurement, legal), and issuing a clear shortlist verdict: **Approve / Revise / Rebuild**.

Typical review output includes:
- Overall score out of 50
- 10 dimension scores with justifications
- Alignment check against the original RFP or RFI (if provided)
- Top strengths and critical issues
- Section-by-section feedback with specific, actionable fixes
- IP and confidentiality audit
- Readability and writing quality assessment
- Argument-quality observations (claims, evidence, logic, trust signals)
- Final shortlist verdict

## When to use it

- Before submitting any bid response or RFI reply
- After Bid Manager generates a draft — Bid Checker is its natural follow-on
- When you want a second opinion on a proposal written outside the bid-manager pipeline
- When a proposal has been through multiple rounds of edits and you've lost sight of whether it still holds together
- When preparing for a shortlist presentation and you want to stress-test the written submission first

## Required inputs

| Input | Details |
|-------|---------|
| **Proposal document** | Drop the proposal into `input/`. Markdown or converted text is ideal; DOCX is accepted and will be converted. |

## Optional / configurable parameters

| Parameter | Default | Notes |
|-----------|---------|-------|
| **Original RFP / RFI** | None | Strongly recommended. If provided, the review includes a requirement-by-requirement alignment check. Drop alongside the proposal in `input/`. |
| **Requirement matrices** | None | If the original RFP includes scoring matrices or evaluation criteria tables, include them. The agent will check alignment against the published weightings. |
| **Focus areas** | Full review across all 10 dimensions | Specify if you want deeper attention on particular dimensions: `writing-quality`, `argument-strength`, `ip-audit`, `executive-summary`, `compliance-section`, `competitive-positioning`, `persuasion` |
| **DOCX export** | Not generated by default | Request `export: docx` if you need the review report in Word format for internal circulation |
| **Evaluator persona emphasis** | Balanced across all five personas | Specify a persona if you know the primary evaluator type: `technical`, `business`, `risk-compliance`, `procurement`, `legal` |

## Full example prompt

```
Please review this bid response before we submit it.

Proposal: input/adia_rfp_response_v3.md
Original RFP: input/adia_rfp_2026-03-20.docx

Focus areas: argument-strength, ip-audit, executive-summary
Evaluator emphasis: risk-compliance, procurement
Export: docx

Context:
- ADIA is a sovereign wealth fund; their evaluation committee is known to include a
  chief risk officer and a procurement director with a legal background
- We have a tight deadline; feedback on the top 3 critical blockers is more useful
  than exhaustive line-by-line notes
- Previous reviewer flagged the integration section as weak — please give it extra scrutiny
```

## ⚠️ Important

- **Send everything in ONE message.** Include the proposal path, any RFP documents, optional matrices, focus areas, and context all at once. Do not send files and context as separate messages.
- **Do not send follow-up messages mid-task.** The agent will confirm when the review report is ready.
- **Scoring is honest, not diplomatic.** The agent will surface real weaknesses. A low score means the proposal needs work before submission — that is the point.
- **Every criticism comes with a fix.** The review does not just flag problems; it tells you what to do about them.
- **Content presence is not enough.** A section that exists but does not persuade will still be scored as deficient. The agent tests argument quality, not just completeness.
- **IP audit is always run.** Internal names, source code references, confidential client names, and competitor mentions are flagged as blockers regardless of which focus areas you specify.
