# Content Updates Queue

## Purpose
Track factual corrections and content improvements identified by the Bid Checker or post-mortem process that need to be applied to content/ sections.

## Queue

| Date | Source | Content File | Issue | Fix Needed | Status |
|------|--------|-------------|-------|------------|--------|
| (populated by post-mortem) | | | | | Pending/Applied/Rejected |

## Rules
1. Factual errors (wrong module count, incorrect API claim, etc.) are Priority 1, fix immediately
2. Missing content (capability not documented) is Priority 2, fix within 1 week
3. Style/quality issues (prose too technical, missing diagrams) are Priority 3, fix during next content refresh
4. Rejected items need a reason (e.g., "actually correct per codebase verification")
